# Preferences

User's coding style and collaboration preferences.

## Coding Style
- ESM imports (`import`/`export`)
- No TypeScript — plain JavaScript
- Arrow functions for route handlers and middleware
- Inline error handling with try/catch in each route
- Comments as section dividers (`// ----- section name -----`)

## Conventions
- JSON responses use `{ ok: true/false, ... }` pattern
- Error responses include `{ error: err.message }`
- Console logging for errors (`console.error`) and key events (`console.log`)
